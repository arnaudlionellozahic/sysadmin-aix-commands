							2.2 Les applis et serveurs d’application :
							------------------------------------------


Ils se trouvent coté présentation uniquement

Pour chaque site, il existe 3 serveurs d’application (dont 2 réellement utilisés et le dernier sert de test) et 3 serveurs IHS (http) associés. 

•	srv_docubase_« site »	et 	https-docubase« 2 lettres du site »

•	srv_webdoc_« site » 		et 	https-webdoc« 2 lettres du site »

•	srv_webexp_« site » 		et 	https-webexp« 2 lettres du site »


Comme pour tous nos serveurs la configuration de nommage des applis reste la même (excepté pour le maroc qui n’est pas standard)

	srv_« appli »_« site »   et   https-« appli »« site »





							2.2.1 Le serveur d’appli « docubase » :
							--------------------------------------


Nom : srv_docubase_*        (Attention pour le maroc son nom est : docubase)


Il contient les applications d’entreprise suivantes :

•	Le « RHEAWEB » :

•	Le « AMFADMIN » :

•	Et le «  CONNECTOR » : qui permet à l'application webdoc d'accéder aux documents stockés dans la base GED

•	Le « AMFCONFIG » : uniquement pour le Maroc



Les URL d’accès : 


Rechercher dans le fichier HTTPD.CONF du site, le « ServerName » : /apps/IBMHTTPServer/WebServer7/https-docubaseSITE/conf/httpd.conf


Attention pour le maroc son nom est : /appli/rheaprint-maroc.rb.echonet/ihs/https-rheaprint-maroc.rb.echonet/conf/httpd.conf


Recherche le fichier avec : 

ps -ef |grep http |grep https-docubase 


exemple:

ps -ef |grep http |grep https-docubase |grep DZ  ------------> ici le "DZ" pour l'algérie


root  360466       1   0   Mar 18      -  3:42 /apps/IBMHTTPServer/WebServer7/bin/httpd -d /apps/IBMHTTPServer/WebServer7 -k start -f 
/apps/IBMHTTPServer/WebServer7/https-docubaseDZ/conf/httpd.conf


cat /apps/IBMHTTPServer/WebServer7/https-docubaseDZ/conf/httpd.conf  -----------> on fait un cat du fichier rechercher pour trouver l'URL

 
<VirtualHost 10.255.35.105:443>  ------> https
ou
<VirtualHost 10.255.35.105:80>   ------> http

        		ServerName rheaalgerie.rb.echonet  ------------------>  ici nous avons l'URL sous le servername  " rheaalgerie.rb.echonet "



Puis, lui rajouter le context suivant en fonction des applis à tester : 


http://« ServerName »/rheaweb/auth     ou   https://« ServerName »/rheaweb/auth    _
										    |

										    |

Exemple :

http://rheaalgerie.rb.echonet/amfadmin

http://rheaalgerie.rb.echonet/connector/cxf
